# Data and Persistence

Purpose: canonical ownership map for player data, formats, backup, and safe
recovery.

Audience: players backing up or troubleshooting data and contributors changing
persistence code.

Status: Maintained and verified against the current path and save code.

Neighboring guides: [player guide](player-guide.md),
[compatibility](compatibility.md), [contributing](contributing.md), and
[roadmap](roadmap.md).

## Path rules

`CONFIG_DIR` below means `$XDG_CONFIG_HOME/dtxmaniars/`, or
`$HOME/.config/dtxmaniars/` when `XDG_CONFIG_HOME` is unset. If neither variable
exists, individual configuration files fall back to the current working
directory. The default song root is also `CONFIG_DIR`; set `DTX_SONG_DIR` to
keep a library elsewhere.

| Data | Default path | Format/version | Write behavior |
|---|---|---|---|
| Settings | `CONFIG_DIR/config.toml` | TOML; no top-level schema field | Direct replacement; missing/unreadable/invalid loads defaults and reports recovery where available |
| HUD/layout | `CONFIG_DIR/layout.toml` | TOML, version 1 | Direct replacement; missing/corrupt loads defaults |
| Keyboard profiles | `CONFIG_DIR/keyboard-profiles.toml` | TOML registry, version 1 | Atomic replacement |
| MIDI profiles | `CONFIG_DIR/midi-profiles.toml` | TOML registry, version 1 | Atomic replacement |
| Lane profiles | `CONFIG_DIR/lane-profiles.toml` | TOML registry, version 1 | Atomic replacement |
| Saved practice loops | `CONFIG_DIR/practice-presets.toml` | TOML registry, version 1 | Atomic replacement after explicit Save as New, Update Saved Loop, or Delete; Last Used updates when Practice starts or continues |
| Legacy bindings | `CONFIG_DIR/bindings.toml` | TOML, version 1 | Migration input; direct replacement only through the legacy API |
| Favorites | `CONFIG_DIR/library-preferences.json` | JSON, version 1 | Direct replacement |
| Native score history | `DTX_SCORES_PATH`, otherwise `./scores.json` | JSON, version 2 | Direct replacement |
| Compatible chart score | `<chart path>.score.ini` | DTXMania-style INI, no schema field | Direct merge/replacement next to the chart |
| Songs/imports | `DTX_SONG_DIR`, otherwise `CONFIG_DIR` | User chart/media files | Import creates a song directory; not application state to reset casually |

“Atomic replacement” means the complete serialized registry is written through
the `dtx-persistence` atomic-file path, with parent creation and overwrite
enabled. “Direct replacement” means `std::fs::write`; it is not claimed to be
crash-atomic. Keep backups before hand-editing or migrating direct-write files.

## Settings, profiles, and migration

`config.toml` owns System, Gameplay, Audio, Drums, and Accessibility settings.
`layout.toml` owns the scene/widget layout and legacy embedded lanes. Once
`lane-profiles.toml` exists, its active lane profile is authoritative.

Keyboard and MIDI profiles are separate so changing one does not rewrite the
other. Built-in profiles are regenerated by the application and user profiles
are persisted. On startup, if a new registry is absent, the application can
migrate the corresponding values from `bindings.toml`. A failed migration
write remains usable in memory for that session; an unreadable or unsupported
registry falls back to built-ins without overwriting the bad file.

The in-app reset flow for keyboard, MIDI, and lane registries creates a
timestamped `.backup-*` file when an existing registry can be preserved. Use
that flow when possible. A newer unsupported version is not silently
downgraded.

Saved practice loops are separate from general settings. Each lookup uses both
the chart's canonical hash and its selected difficulty index, so a loop saved
for one difficulty is not offered for another. The registry retains the source
path as metadata; the current UI does not display it or use it for recovery.
Editing a loaded loop changes a session draft; it is not persisted until the
player chooses Save as New or Update Saved Loop.
Deleting a saved loop is also explicit. Last Used is the one automatic snapshot
per chart and difficulty, written only when Start Practice or Continue Practice
commits the draft.

If `practice-presets.toml` is missing, Practice starts with an empty registry. A
corrupt, unreadable, or newer-version file is preserved and the registry stays
read-only for that session rather than overwriting recoverable data.

## Scores and qualification

The native score store path is intentionally independent of `CONFIG_DIR`:

```sh
DTX_SCORES_PATH=/absolute/path/to/dtxmaniars-scores.json dtxmaniars
```

Without that variable, `scores.json` is relative to the process working
directory. Launching the game from different directories can therefore appear
to produce different histories. Set an absolute path for predictable installed
use.

An ordinary qualifying result is normal play at exactly `1.00x` with Standard
fail mode. Practice and modified Play Speed runs do not append native history
or update compatible score records. At `1.00x`, No Fail runs append assisted
`no_fail` entries to native history, but the game excludes them from personal
bests and player skill and never writes them to `<chart>.score.ini`. Ordinary
qualifying results merge best/last/play/clear data into `<chart>.score.ini`;
the chart directory must be writable. The scoring layer can read and import
existing DTXManiaNX `.dtx.score.ini` data.

Native JSON and chart-side INI are complementary. Removing `scores.json` does
not remove chart-side files, and removing `.score.ini` files does not remove
native history already stored in JSON.

## Backup

With the game closed, preserve:

1. the entire `CONFIG_DIR`, including imported songs if it is also the song
   root and including `practice-presets.toml` for saved practice loops;
2. the file named by `DTX_SCORES_PATH`, or the launch directory's
   `scores.json`;
3. external `DTX_SONG_DIR` libraries, including chart-side `.score.ini` files.

Copying the whole directories preserves profile names, active selection,
favorites, relative media, and chart-side history. Restore into the same paths
or launch with the same environment overrides.

## Safe reset and recovery

- Invalid `config.toml`: close the game, rename the file, and relaunch. Defaults
  load; copy desired values into the newly saved file later.
- Invalid `layout.toml`: rename it to restore the default scene. If a lane
  registry exists, it remains authoritative for lanes.
- Invalid profile registry: preserve the original, use the in-app reset/backup
  action, then recreate or copy the affected user profile.
- Invalid `practice-presets.toml`: preserve or rename the original before
  recovery. Practice treats it as read-only for that session; do not overwrite
  it or edit its version number to force a load.
- Missing favorites: deleting `library-preferences.json` safely resets only
  favorites, but the change is irreversible without a backup.
- Missing native scores: confirm the launch directory and
  `DTX_SCORES_PATH` before assuming data was lost.
- Unwritable chart directory: normal native history may save while the
  chart-side `.score.ini` update fails. Fix ownership/permissions; do not run
  the game as a privileged user.
- Unsupported newer version: keep the file and upgrade the application. Do not
  edit the version number to force an older reader.

Never delete `CONFIG_DIR` as a blanket reset until checking whether it is also
the only copy of the song library. Never edit or use the repository's
`references/` tree as player data.
